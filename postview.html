@chaewon

Go creates native binaries - you don't need an interpreter or runtime to run them. So yes, you can just compile on your home machine (cross compile if the target computer is a different platform) and then just run the binary on the target computer (that's assuming that the machine isn't locked down in a way that disallows executing arbitrary binaries - but if that's the case, you're just plain out of luck).
